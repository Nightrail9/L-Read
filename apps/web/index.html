从上传文献到产出笔记的完整流程

快速开始（1 分钟）

先在「模型配置」中完成连通性检测，再上传 PDF，点击“开始分析文献”，系统将自动生成多维笔记。

步骤 1：配置模型 步骤 2：上传论文 步骤 3：查看笔记

标准任务流程

STAGE 01

创建任务并上传 PDF

支持拖拽上传，文件建议不超过 50MB。

STAGE 02

可选关联代码仓库

支持 GitHub URL 或本地路径，系统自动构建索引后进行对齐分析。

STAGE 03

阅读与编辑笔记

支持预览、编辑、复制与下载 Markdown，并可单模块重试。

项目列表与同步说明

  • 项目列表优先展示后端任务记录；页面刷新或重新打开时会自动拉取后端数据。
  • 点击「同步项目」会触发目录与数据库的重新对齐，适用于迁移或补录历史任务。
  • 若同步后仍无数据，请先确认后端服务正常运行，再检查数据目录与任务产物是否存在。

常见问题

为什么“开始分析文献”按钮是灰色？

通常是未上传 PDF、已有任务正在运行，或系统仍在处理中。上传文件后按钮会自动激活。

模块部分失败后怎么处理？

在项目列表点击“重试”，系统会优先补跑失败模块并保留已成功结果。

笔记可以导出或二次编辑吗？

支持。你可以在阅读页复制 Markdown、下载本地文件，或在编辑器中修改后直接保存回后端产物。

删除该项目？

删除后无法恢复。
